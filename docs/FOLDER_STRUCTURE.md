## File Responsibility Rules

| File/Folder | What goes here | What does NOT go here |
|---|---|---|
| `main.dart` | App init, Firebase init, ProviderScope, GoRouter | Any UI or logic |
| `services/` | Raw Firebase SDK calls | Business logic, UI |
| `features/*/data/` | Repository impl, models, mappers | UI, providers |
| `features/*/domain/` | Interfaces, entities | Firebase, UI |
| `features/*/presentation/screens/` | Screen scaffold only | Business logic |
| `features/*/presentation/widgets/` | Single-purpose UI components | State, logic |
| `features/*/presentation/providers/` | Riverpod providers | UI, Firebase calls |
| `theme/` | Design tokens only | Logic, widgets |
| `shared/widgets/` | Reusable across 2+ features | Feature-specific UI |

---

## New Feature Checklist

When adding any new feature, create in this order:
1. `domain/` вЂ” interface + entity
2. `data/` вЂ” model + repository implementation
3. `services/` entry (if new Firebase collection needed)
4. `providers/` вЂ” Riverpod providers
5. `widgets/` вЂ” UI components
6. `screens/` вЂ” assemble components
7. Add route to `app_router.dart`
